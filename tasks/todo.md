# Opencode Task Management

## Task Planning
- [ ] Write detailed plan for non-trivial tasks (3+ steps)
- [ ] Include architectural decisions and verification steps
- [ ] Create checkable items for each task

## Task Execution
- [ ] Verify plan before starting implementation
- [ ] Track progress by marking items complete
- [ ] Provide high-level summary of changes at each step
- [ ] Document results and add review section

## Lesson Capture
- [ ] Update lessons after any user corrections
- [ ] Write preventive rules for mistakes
- [ ] Review relevant lessons at session start

## Verification
- [ ] Never mark task complete without proof
- [ ] Run tests and check logs
- [ ] Diff behavior when relevant
- [ ] Ask: "Would a staff engineer approve this?"
